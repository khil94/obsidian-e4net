
## UI 피드백

- 연속 접속, 학습일, 평균 학습 문제 등 삭제
	- 기본 제공 문제가 적고 모의고사 유료화를 염두에 둔 상황이라면 그다지 필요 없는 정보로 보임	
- 학습 온도가 무엇인지 알려주는 정보 버튼 추가 제안
	- '학습 온도'의 기준이 직관적으로 와닿지 않습니다. 우측 상단에 툴팁이나 정보(i) 아이콘을 추가하여 산정 방식을 안내해주면 좋겠습니다.
- By type 섹션
	- 언어의 혼용 존재. (다른 곳은 한글 title, 이곳은 영문)
	- 우측의 xx문제 문구가 이쁘게 보이지 않음
		- 'By type' 우측의 퍼센트(%)와 문제 수(xx문제) 텍스트 간의 위계와 정렬이 다소 어색합니다. 
		- 퍼센트를 강조하고 문제 수는 게이지 하단으로 작게 빼거나 툴팁으로 제공하여 UI를 더 깔끔하게 가져가는 것을 제안합니다.
	- 전반적으로 보기 좋다
- 학습 내역 섹션
	- 주간과 월간이 같은 학습 내역 UI를 공유
		- '주간/월간' 탭 전환 시 학습 내역(잔디밭) UI가 동일하게 유지되는 것이 어색합니다. 
		- 주간 탭에서는 7일 치 막대그래프를, 월간 탭에서는 현재의 잔디밭이나 달력 형태를 제공하여 필터의 컨텍스트에 맞는 데이터 시각화가 필요합니다.
- 주간/월간 필터의 적용 범위 명확화
	- 상단 주간/월간 토글 버튼을 눌렀을 때, 하단의 모든 데이터가(푼 문제, 정답률 등) 주간/월간 기준으로 변경되는 것인지, 아니면 일만 변하는 것인지 UI로 인지하기 어려운 문제.
	- 필터가 적용되는 영역을 시각적으로 묶어주거나 명확한 안내가 필요
- 페이지 전반
	- 페이지 전반의 디자인 퀄리티는 훌륭하지만, 앱 전체의 테마와 본 페이지(통계)와의 테마 일관성이 부족함. (다크모드)
	- 앱 방향성이 모의고사를 기반으로한 특정 사용자층을 겨냥한 전문화된 앱이라면 필요한 기능으로 보여짐

## 기능 제안

- 모의고사가 주 수익화 컨텐츠인 이상 많은 모의고사 문를 제공함과 동시에 차별화된 추가 서비스를 도입하는 것을 고려해볼 만 함
	- 문제 유형 카테고리가 존재한다면( 읽기, 듣기가 아닌 비문학, 문학, 빈칸 넣기 등 )
		- 유형별 정답률
			- 사용자가 본인이 약한 유형 파악에 도움
		- 유형별 풀이시간, 시간 초과 등... 
	- 다만 이렇게 많은 기능을 넣으면 바텀 시트 형태가 아니라 별개의 페이지화 되어야 할 것으로 보임
