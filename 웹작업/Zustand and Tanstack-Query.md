
## 개요

사용자 데이터 관리에 있어서 도입해야 할 라이브러리를 선정하고자 한다.

## Zustand

전역 상태 관리 라이브러리. 사용자 로그인 정보 등을 전역으로 관리하기 위해 사용할 수 있다. 그러나

- 현재 사용자 기본 정보는 sessionStorage에 저장된 상황
- 또 사용자 크레딧과 같은 정보가 사용자 정보에 귀속된 상황
	- 크레딧 같은 서버 정보는 tanstack-query로 관리하는 것이 맞다.
	- 필요성 감소

## Tanstack-query 

서버 상태를 관리하기 위한 라이브러리로, 상시 여러 곳에서 보여줘야 하며 신선도를 유지해야 하는 사용자 크레딧 정보가 사용자 정보에 종속되어 있으므로, 사용자 정보를 tanstack-qeury로 관리하는 것이 적절할 것으로 보인다.

## 결론

zustand + tanstack-query 방식에서 tanstack-query only
